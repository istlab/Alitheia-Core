Adding Projects to the System

In order to allow their analysis, the system should allow to create projects.

The system can create empty projects and add them to its projects list. Such a project need to know where the mail activity, bug activity and source repository of the project are.

Example

Given the system only having the initial empty project list, when we add the new project "test project", then the project list becomes:

Project Name
test project

Once this new project is added, when we try to add it again, the project list is not modified:

Project Name
test project
